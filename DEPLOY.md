# Инструкция по развертыванию на GitHub Pages

## Шаги для публикации сайта

1. **Создайте репозиторий на GitHub**
   - Перейдите на https://github.com/new
   - Создайте новый репозиторий (например, `npai-platform`)

2. **Добавьте remote и отправьте код**
   ```bash
   git remote add origin https://github.com/[ваш-username]/[название-репозитория].git
   git branch -M main
   git push -u origin main
   ```

3. **Настройте GitHub Pages**
   - Перейдите в Settings репозитория
   - В левом меню выберите "Pages"
   - В разделе "Source" выберите:
     - Branch: `main`
     - Folder: `/landing-site`
   - Нажмите "Save"

4. **Дождитесь публикации**
   - GitHub автоматически опубликует сайт
   - Обычно это занимает 1-2 минуты
   - Сайт будет доступен по адресу: `https://[ваш-username].github.io/[название-репозитория]/`

## Обновление сайта

После внесения изменений:

```bash
git add .
git commit -m "Описание изменений"
git push
```

GitHub Pages автоматически обновится через несколько минут.

## Проверка локально

Перед публикацией проверьте сайт локально:

```bash
cd landing-site
python3 -m http.server 8000
```

Откройте http://localhost:8000 в браузере.

## Важные замечания

- Убедитесь, что все пути к файлам относительные (не начинаются с `/`)
- Проверьте, что все изображения загружены
- Убедитесь, что все ссылки работают
- Проверьте мобильную версию сайта
